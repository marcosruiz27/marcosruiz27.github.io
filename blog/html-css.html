Difference between relative, absolute and fixed position in CSS

There are several options to position an element using CSS. These are the 3 most used:

Relative Positioning allows an element to be adjusted from its normal position. When you set the top, right, bottom, and left properties of an element with relative positioning, it moves from its original location. However, it’s important to note that other elements on the page will not fill the gap left by the moved element. Also, the properties can carry a positive or negative numerical value.

Absolute Position adjusts an element’s position relative to its parent (you need two elements, which are the parent and child elements). If no parent element is present, the document body will be used as the parent. You can use the same properties as Relative Positioning. Important: The position of the parent element should be set as a relative value. If not, the child element will be offset relative to the page.

Fixed Position keeps an element in the same place even when the page is scrolled. To position the element, we use top, right, bottom, and left properties. The fixed positioning offsets elements according to the browser page. It is like absolute positioning, but the parent element is the viewport (browser page.)
