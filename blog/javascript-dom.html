Javascript

Javascript Analogy

In the final reflections of Sprint Two I compared a website to a house. You use HTML for the structure of the house and then CSS to decorate the rooms and exterior. To introduce the concept of Javscript into this analogy, we have to think of this house as a smart house, one that performs complex functions; like turning on lights automatically if it's dark outside, adjusting the temperature depending on the season, checking what produce need to be restocked in the fridge, reminding people of appointments when they are having breakfast. Javascript is connected to both HTML and CSS in order to deliver these functions. All these functions that make the house smart and interactive, would be thanks to Javascript. As Javascript makes websites smart and interactive.

Control Flow and Loops

Control Flow is the normal order in which a series of actions are performed to achieve a purpose. Despite having an usual order, this might be altered depending on the result of some actions. For example to start my working day this would be the control flow:

  1. Turn on the computer
  2. Insert password
  3. Open Slack
  4. Say "Good morning" to my team

However, if after completing the action number 2, the message is "Incorrect password" the control flow would be the following:

  1. Turn on the computer
  2. Insert password
  3. Check and insert correct password
  4. Open Slack
  5. Say "Good morning" to my team

Loop is a series of actions that we repeat until a condition is met. For example, when we are cooking we add salt and condiments until food tastes the way we like. If it's not good enough we will continue adding salt and spices until it tastes good. The actions would be addidng salt and condiments and the condition would be that food tastes like we like.

DOM

DOM stands for Document Object Model. It is a programming interface for web documents. It represents the page so that programs can change the document structure, style, and content. Having said that, if we go back to the smart house analogy, we can think of DOM as a document where information about the structure, style and functions of the house exist. It contains every single detail of the house and it is accessible so we can make adjustments that will be reflected in the house. For example, let's suppose we have a table with id "table77" in a website. Users are complaining saying that the table is hard to read due to the size and type of fonts. I can access the DOM via browser console and using "document.getElementById" I can access the table and try different font sizes and types, until I see one that is accepatble to most users.

Difference Accessing Arrays and Objects

Both elements are useful to store data, but they are used in different ways and accessed in different ways. Arrays are ordered collections of elements, typically accessed using an index (which is a numerical position).

For example: let names = ['marcos', 'albert', 'simon'];

The index is zero-based, meaning the first element is at index 0

The output of console.log(names[2]); would be simon.

Functions

Functions are pieces of code that are designed to perform a task like storing data or make a calculations. They normally take input, then process it based on what we ask them to do and then provide and output. Functions are really helpful because we can just write it once and then use it many times in the code. They also allow modularity in the code, which means we can use them to break down complex code into smaller parts, making it easy to manage, troubleshoot and test.
